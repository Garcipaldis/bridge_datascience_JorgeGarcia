Taller iniciación con Dash¶

En este taller guiado vas a crear tu primera aplicacin en Dash. Desplegars un dashboard en local como si de una pgina web se tratase. Vas a utilizar la base de datos de Aplicaciones Android, donde tienes todo el listado de apps del Play Store, con su ranting, cantidad de reviews, descargas, tamao...

Empezars tu Hola Mundo creando unas primeras grficas, despus filtros y configurars su interaccin con las grficas.

Te dejo el enlace a la pgina oficial de la documentacin.

Instalación¶

La instalacin de Dash se realiza como cualquier otro paquete de Python, abriendo un terminal y escribiendo el comando:

pip install dash

Introducción¶

Las aplicaciones Dash se componen de dos partes. Por un lado estar el layout, que describe cmo es la aplicacin. Y por otro lado la interactividad entre los componentes de la app.

Adems, cada elemento de la app es un componente de Dash. Los componentes los puedes encontrar en las libreras dash_core_components y dash_html_components (componentes puros HTML). Puedes crear los tuyos propios mediante JaveScript o React.js.

Entorno¶

Una app de Dash es una pgina web. Se programa en Python, pero despus el propio Dash crea por debajo el rederizado en HTML, para que podamos visualizar e interactuar con la app desde un navegador. Por tanto, no deja de ser un servicio que est corriendo en nuestro ordenador local(de momento), al que accederemos desde el navegador.

Tendrs que programar un script de Python que por conveniencia se llama app.py, y dejar corriendo ese script, como si fuese un servidor. Una vez est corriendo, realizaremos consultas a ese servicio a travs del navegador.

NOTA: Tienes que programar un script (.py). No te vale un Notebook de Jupyter.

Creación proyecto¶

Crea un proyecto en Pycharm (o en IDE que ests usando), vaco, y aade un script que se llame app.py.

Primera app¶

Vamos a hacer nuestro primero Hola Mundo en Dash. Para ello copia el siguiente cdigo en el script app.py, y corre el script (NO correr lnea a lnea). Veamos qu significa este cdigo:

  • Importacin de mdulos de Dash
  • Estilos: Estamos utilizando una plantilla de estilo css para nuestra app.
  • Lectura de datos: leemos el CSV de apps de Android en un dataframe
  • fig: antes de meternos en el layout de la app, dejamos declarada la grfica que mostraremos luego
  • app.layout: Aqu dentro va toda la app. Es un arbol de componentes. Si te fijas, consta de un html.Div, y dentro una lista de componentes (children).
  • Componentes: Una cabecera grande, un subttulo y el grfico. Son declarativos, puedes describir los componentes a travs de sus atributos.
  • main: es el ejecutable del script. En este punto cambiaremos las configuraciones de la ejecucin en el servidor. De momento no tocamos nada
# visit http://127.0.0.1:8050/ in your web browser.

import dash
import dash_core_components as dcc
import dash_html_components as html
import plotly.express as px
import pandas as pd

external_stylesheets = ['https://codepen.io/chriddyp/pen/bWLwgP.css']

app = dash.Dash(__name__, external_stylesheets=external_stylesheets)

df = pd.read_csv("data/googleplaystore.csv")

fig = px.histogram(df, x="Rating", range_x=[0.8, 5.2])

app.layout = html.Div(children=[
    html.H1(children='Dashboard aplicaciones Android'),

    html.Div(children='''
        Primera app de aprendizaje con Dash
    '''),

    dcc.Graph(
        id='example-graph',
        figure=fig
    )
])

if __name__ == '__main__':
    app.run_server(debug=True)

Hot-Reloading¶

Se trata de una funcionalidad de Dash que permite modificar la app en cdigo, y que se actualice automticamente (tarda unos segundos) en el navegador. Muy til ya que no tenemos que parar el servicio y volver a ejecutarlo para visualizar los cambios.

Estando en ejecucin la app, prueba a cambiar la cabecera, y a continuacin refresca el navegador.

Estilos de componentes¶

Los componentes tienen el atributo de estilo, donde podremos modificar el color de la letra, del fondo o tamao del texto. Funciona por tags, como HTML. Dentro del estilo hay unos atributos (textAlign, color...) que podrs modificar.

Prueba a crear otro sript app2.py con el siguiente cdigo, ejectalo y visualiza el output en el navegador.

# visit http://127.0.0.1:8050/ in your web browser.

import dash
import dash_core_components as dcc
import dash_html_components as html
import plotly.express as px
import pandas as pd

external_stylesheets = ['https://codepen.io/chriddyp/pen/bWLwgP.css']

app = dash.Dash(__name__, external_stylesheets=external_stylesheets)
colors = {
    'background': '#111111',
    'text': '#7FDBFF'
}

df = pd.read_csv("data/googleplaystore.csv")

fig = px.histogram(df, x="Rating", range_x=[0.8, 5.2])
fig.update_layout(
    plot_bgcolor=colors['background'],
    paper_bgcolor=colors['background'],
    font_color=colors['text']
)

app.layout = html.Div(
    style={'backgroundColor': colors['background']},
    children=[
    html.H1(children='Dashboard aplicaciones Android',
            style={
                'textAlign': 'center',
                'color': colors['text']
            }
            ),

    html.Div(children='''
                Primera app de aprendizaje con Dash
            ''',
             style={
                 'textAlign': 'center',
                 'color': colors['text']
             }
             ),

    dcc.Graph(
        id='example-graph',
        figure=fig
    )
])

if __name__ == '__main__':
    app.run_server(debug=True)

Graph¶

Dentro de los dash_core_components, tienes Graph, que es el encargado de renderizar grficas interactivas utilizando la librera de Plotly. El argumento figure que utiliza Graph es el mismo que utilizbamos con Plotly.

Olvida los estilos del apartado anterior, y retoma la primera app para crear otra nueva. Vemos cmo son las distribuciones del rating por categora de app.

# visit http://127.0.0.1:8050/ in your web browser.

import dash
import dash_core_components as dcc
import dash_html_components as html
import plotly.express as px
import pandas as pd

external_stylesheets = ['https://codepen.io/chriddyp/pen/bWLwgP.css']

app = dash.Dash(__name__, external_stylesheets=external_stylesheets)

df = pd.read_csv("data/googleplaystore.csv")

fig = px.histogram(df, x="Rating", range_x=[0.8, 5.2])

fig2 = px.box(df, x="Category", y="Rating", color="Category", range_y=[0.8, 5.2])

app.layout = html.Div(children=[
    html.H1(children='Dashboard aplicaciones Android'),

    html.Div(children='''
        Primera app de aprendizaje con Dash
    '''),

    dcc.Graph(
        id='example-graph',
        figure=fig
    ),
    dcc.Graph(
        id='example-graph-box',
        figure=fig2
    )
])

if __name__ == '__main__':
    app.run_server(debug=True)

Core components¶

Dash tiene una gran cantidad de componentes propios, como menus desplegables, radio buttons, grficos, entre otros. Tambin, como en el caso de los componentes HTML, son declarativos, por lo que tendrs que ir configurando sus atributos. Aqu tienes todos los componentes disponibles

Vamos a empezar a colocar algn filtro en nuestro dashboard. En concreto, un selector desplegable con multiopcin, que nos permita seleccionar entre las apps gratuitas, o las de pago. Insertamos el siguiente componente entre los ttulos y las grficas:

dcc.Dropdown(
        id='id_free_paid',
        options=[
            {'label': 'Free', 'value': 'Free'},
            {'label': 'Paid', 'value': 'Paid'}
        ],
        value=['Free', 'Paid'],
        multi=True
    )

De momento tenemos una caja que tiene un menu desplegable donde seleccionamos Free y/o Paid, y no hace nada ms. La inteligencia del filtrado la introduciremos con un callback.

Callbacks¶

Son funciones que se llaman automticamente cuando cambia algn componente de la app. Para qu se usa esto? Para poner botones, interacciones del usuario con la web y filtrar las grficas en funcin de esas interaccciones.

Como ya tenemos un selector, ahora vamos a vincularlo con las otras dos grficas usando un callback. Coloca este codigo entre el layout y el main. Tienes que importar los siguients componentes:

from dash.dependencies import Input, Output, State

@app.callback(
    Output('example-graph', 'figure'),
    Output('example-graph-box', 'figure'),
    [Input('id_free_paid', 'value')])
def update_graphs_selector(selected_type):
    filtered_df = df[df["Type"].isin(selected_type)]

    fig = px.histogram(filtered_df, x="Rating", range_x=[0.8, 5.2])
    fig2 = px.box(filtered_df, x="Category", y="Rating", color="Category", range_y=[0.8, 5.2])

    fig.update_layout()
    fig2.update_layout()

    return fig, fig2

Lo que va antes de la funcin es qu inputs hay y sobre qu outputs van a afectar. Para vincularlos utilizamos sus ids. El argumento de la funcin update_graphs_selector es el input del selector (una lista). Con esa lista filtramos los datos del dataframe, creamos figuras nuevas con estos datos y refrescamos el front con esas figuras.

Posicionamiento en layout¶

Hasta ahora hemos estado creando grficas y apilndolas verticalmente. Pero, y si queremos aplicarlas horizontamente? Ir haciendo cajas verticales y horizontales con diferentes componentes, tal y como se hace en HTML. Para ello, nos vamos a apoyar en el paquete dash_bootstrap_components. Tienes que instalarlo mediante:

pip install dash_bootstrap_components

Estos componentes son responsive, por lo que se adaptan dependiendo del tamao de la pantalla, por lo que dar igual si la pgina se abre desde un mvil, tablet u ordenador. Te recomiendo que leas la documentacin para que comprendas mejor cmo colocar tus componentes en el layout. Para alinear tus componentes como quieras, tendrs que jugar con:

  • dbc.Container
  • dbc.Row
  • dbc.Column

El siguiente ejemplo est compuesto de:

  1. Dos Rows para los ttulos
  2. Otro Row en el que irn dos columnas:
    • Un grfico que ocupa todo el alto de la fila
    • Un Col con dos grficos
  3. Un Row con tres Cols, en cada uno un grfico de diferente color.

NOTA: Adems de instalar el nuevo paquete, fjate que cuando se crea el objeto app, utilizamos un elemento del nuevo mdulo instalado dash_bootstrap_components

# visit http://127.0.0.1:8050/ in your web browser.

import dash
import dash_core_components as dcc
import dash_html_components as html
import plotly.express as px
import pandas as pd
import dash_bootstrap_components as dbc

external_stylesheets = ['https://codepen.io/chriddyp/pen/bWLwgP.css']

app = dash.Dash(__name__, external_stylesheets=[dbc.themes.GRID, external_stylesheets])

df = pd.read_csv("data/googleplaystore.csv")

fig_blue = px.histogram(df, x="Rating", range_x=[0.8, 5.2], color_discrete_sequence=['blue'])
fig_red = px.histogram(df, x="Rating", range_x=[0.8, 5.2], color_discrete_sequence=['red'])
fig_orange = px.histogram(df, x="Rating", range_x=[0.8, 5.2], color_discrete_sequence=['orange'])
fig_black = px.histogram(df, x="Rating", range_x=[0.8, 5.2], color_discrete_sequence=['black'])
fig_green = px.histogram(df, x="Rating", range_x=[0.8, 5.2], color_discrete_sequence=['green'])

fig_box = px.box(df, x="Category", y="Rating", range_y=[0.8, 5.2])
fig_box2 = px.box(df, x="Category", y="Rating", color="Category", range_y=[0.8, 5.2])


app.layout = html.Div(
        [
            dbc.Row(dbc.Col(html.H1(children='Dashboard aplicaciones Android'))),
            dbc.Row(dbc.Col(html.Div(children='Primera app de aprendizaje con Dash'))),
            dbc.Row(
                [
                    dbc.Col(dcc.Graph(
                        id='graph-blue',
                        figure=fig_blue,
                        style={'height': '100%'}
                     )),

                    dbc.Col([
                            dbc.Row(dcc.Graph(
                                id='graph-box',
                                figure=fig_box
                            )),
                            dbc.Row(dcc.Graph(
                                id='graph-box2',
                                figure=fig_box2
                            ))]
                    )
                ]
            ),

            dbc.Row(
                [
                    dbc.Col(dcc.Graph(
                        id='graph-orange',
                        figure=fig_orange
                    )),
                    dbc.Col(dcc.Graph(
                        id='graph-black',
                        figure=fig_black
                    )),
                    dbc.Col(dcc.Graph(
                        id='graph-green',
                        figure=fig_green
                    ))
                ]
            )
        ]
)



if __name__ == '__main__':
    app.run_server(debug=True)
